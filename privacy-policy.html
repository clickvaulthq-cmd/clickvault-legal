Privacy Policy

Effective date: March 2026  ·  ClickVault Extensions  ·  PasteGuard v0.1.0

Plain English Summary

  • PasteGuard never reads your clipboard unless you actively press paste.
  • Your clipboard contents are never transmitted to any server, ever.
  • No analytics, no telemetry, no third-party tracking of any kind.
  • The only data stored is your settings (on/off, trusted domains) — locally in your browser.
  • You can uninstall and all data is gone instantly.

1. Who we are

PasteGuard is a Chrome browser extension developed by ClickVault Extensions. It is a standalone tool with no user accounts, no backend servers, and no cloud infrastructure required for its core function.

Contact: support@clickvault.io

2. What data PasteGuard accesses

PasteGuard accesses clipboard content only at the exact moment you perform a paste action (Ctrl+V or equivalent) in a browser tab. This access is read-only, instantaneous, and never stored.

The content is checked against detection patterns locally in your browser. The result of that check (block or allow) is determined entirely on your device. The clipboard content itself is discarded immediately after the check.

Data When accessed Stored? Transmitted?
Clipboard text Only during a paste event Never Never
Detection result Immediately after paste check Block count only Never
Extension settings On extension load and change Yes — locally Never
Trusted domains When you add a trusted site Yes — locally Never

3. What data is stored locally

PasteGuard uses chrome.storage.sync to persist the following settings across browser sessions:

  • Whether protection is enabled or disabled
  • Whether strict mode is enabled
  • Whether to skip password fields
  • Your list of trusted domains
  • A running count of pastes blocked (number only, no content)

This data is stored in Chrome's built-in sync storage. If you are signed into Chrome, Google may sync this settings data across your devices per Google's own privacy policy. PasteGuard itself has no access to this synced data beyond reading and writing your settings.

No clipboard content, no paste history, and no sensitive values are ever written to storage.

4. What PasteGuard never does

Hard guarantees

  • Never reads your clipboard in the background or proactively
  • Never transmits clipboard content to any external server
  • Never sends analytics, usage data, or telemetry
  • Never injects third-party scripts or loads remote code
  • Never creates a user account or requires login
  • Never accesses passwords, form data, or browser history
  • Never sells, shares, or monetises any user data

5. Permissions explained

PasteGuard requests the following Chrome permissions:

  • storage — Used to save your settings and trusted domains list locally in Chrome.
  • host_permissions: <all_urls> — Required so that PasteGuard can intercept paste events on every website you visit. Without this permission, the extension could only protect a manually specified list of sites. Clipboard protection that only works on some sites is not real protection. This permission does not grant access to your browsing history, tab contents, or any page data — it only enables the paste event listener.

6. Third parties

PasteGuard has no third-party integrations, SDKs, or analytics libraries. The extension makes zero outbound network requests during normal operation.

If you purchase a paid licence, payment is processed by Stripe via ExtensionPay. That transaction is handled entirely by Stripe's infrastructure and is subject to Stripe's privacy policy. PasteGuard does not receive or store your payment details.

7. Children's privacy

PasteGuard is a developer tool intended for adults. We do not knowingly collect any data from children under 13. If you believe a child has provided personal information, please contact us and we will address it immediately.

8. Changes to this policy

If we make material changes to this privacy policy, we will update the effective date at the top of this page and, where appropriate, notify users via the Chrome Web Store listing update notes.

Continued use of PasteGuard after a policy update constitutes acceptance of the revised terms.
